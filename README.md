# Trauma-Former: Real-time prediction of trauma-induced coagulopathy using an inverted transformer

[![Python 3.10](https://img.shields.io/badge/python-3.10-blue.svg)](https://www.python.org/downloads/release/python-310/)
[![PyTorch](https://img.shields.io/badge/PyTorch-2.1-orange.svg)](https://pytorch.org/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
This repository contains the complete source code for the paper:

> **Real-time prediction of trauma-induced coagulopathy using an inverted transformer (Trauma-Former): a methodological feasibility and simulation study based on the ADEMP framework**  
> Xiaolei Huang, Wenliang Chen, Guan Wei, Wenjia Lin  
> *BMC Medical Research Methodology* (under review)  

**Trauma-Former** is an **inverted Transformer (iTransformer)** architecture designed for real-time prediction of trauma-induced coagulopathy (TIC) from continuous 1 Hz vital sign streams (HR, SBP, DBP, SpO₂). The model treats the entire 60‑second history of each variable as an independent token and applies multi‑head self‑attention **across variables**, directly modeling the pathophysiological coupling (e.g., heart rate vs. pulse pressure) that characterizes compensated hemorrhagic shock.

This repository provides:

- A **physiologically informed synthetic data generator** based on a multivariate Ornstein–Uhlenbeck process, parameterized using published reference ranges (no real patient data used).
- The **Trauma-Former** model implementation in PyTorch.
- **Five baseline models**: LSTM, XGBoost, PatchTST, Informer, and the shock index.
- **Complete training and evaluation pipelines** with patient‑level 5‑fold cross‑validation.
- **Robustness tests** against Gaussian noise, missing data, and sensor failures.
- **Network latency simulations** for 5G URLLC and 4G LTE.
- **All visualization scripts** to reproduce every figure and table in the paper.

The code is released under the MIT license to promote open science and reproducibility. All results reported in the manuscript can be regenerated by following the instructions below.

## 🚀 Quick Start

### 1. Clone the repository
```bash
git clone https://github.com/DoctorLin1990/trauma-former.git
cd trauma-former
```
### 2. Set up the environment
Create and activate a virtual environment (recommended), then install dependencies:
```bash
# Create virtual environment
python -m venv venv
# Activate it (Linux/macOS)
source venv/bin/activate
# For Windows: .\venv\Scripts\activate

# Install required packages
pip install -r requirements.txt
```
### 3. Generate the synthetic datasets
Run the data generation script to create the development cohort (1,240 episodes, 50% TIC) and the independent test set (1,000 episodes, 25% TIC):
```bash
python data/generate_datasets.py
```
This will save the datasets as .npy files in the data/ folder.
Alternatively, you can download the pre‑generated datasets from [Zenodo link to be added upon acceptance] to skip this step.

### 4. Train and evaluate Trauma-Former (5‑fold cross‑validation)
```bash
python experiments/run_cv.py --config configs/trauma_former.yaml
```
Results will be saved in results/logs/ and summary metrics will be printed.

### 5. Reproduce paper figures
All figures can be generated using the provided Jupyter notebooks:
```bash
jupyter notebook notebooks/
``` 
See the Replicating paper results section for the mapping between notebooks and figures.

🧠 Using the code with your own data
While this repository is designed for synthetic data, you can adapt it to real‑world datasets:

Replace the data loader: Modify data/dataset.py to read your own CSV files (must contain HR, SBP, DBP, SpO₂ at 1 Hz). Ensure the label (TIC status) is provided per time window.

Adjust preprocessing: You may need to change normalization or missing data handling in data/preprocessing.py. Our implementation assumes data is already resampled to 1 Hz.

Re‑run experiments: Use the same training and evaluation scripts.

If you adapt this code for a new study, please cite our paper.
